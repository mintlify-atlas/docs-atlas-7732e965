## Overview

The ETI (Ética de la Investigación) service manages ethics review processes for research projects:

- **Comités** - Ethics committees
- **Memorias** - Ethics reports and submissions
- **Evaluaciones** - Ethics evaluations and reviews
- **Peticiones de Evaluación** - Evaluation requests

Base URL: `/comites`, `/memorias`, `/evaluaciones`

## Authentication

All endpoints require JWT authentication with ETI-specific permissions:
- `ETI-MEM-*` - Memoria (report) permissions
- `ETI-EVC-*` - Evaluacion permissions
- `ETI-ACT-*` - Acta (minutes) permissions

<CodeGroup>
```bash GET /comites

  -H "Authorization: Bearer {token}"
```
</CodeGroup>

<ResponseField name="content" type="array">

</CodeGroup>

### Get Committee Memorias

<CodeGroup>
```bash GET /comites/{idComite}/memorias-peticion-evaluacion/{idPeticionEvaluacion}
curl -X GET "https://api.example.com/comites/1/memorias-peticion-evaluacion/123" \
  -H "Authorization: Bearer {token}"
```
</CodeGroup>

**Required Permission:** `ETI-PEV-INV-C`, `ETI-PEV-INV-ER`

<CodeGroup>

### Create Memoria

<CodeGroup>
```bash POST /memorias
curl -X POST "https://api.example.com/memorias" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "titulo": "Research Ethics Assessment",
    "comite": {"id": 1},
    "peticionEvaluacion": {"id": 456},
    "requiereRetrospectiva": false
  }'
```
</CodeGroup>

**Required Permission:** `ETI-MEM-INV-CR`, `ETI-MEM-INV-ER`

### Update Memoria

<CodeGroup>

```bash PUT /memorias/{id}
curl -X PUT "https://api.example.com/memorias/123" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
```

## Common Patterns

### Pagination

All list endpoints support pagination:

<ParamField query="page" type="integer" default="0">
  Page number (zero-based)
</ParamField>

<ParamField query="size" type="integer" default="10">
  Number of items per page
</ParamField>

<ParamField query="sort" type="string">
  Sort specification (e.g., `numReferencia,desc`)
</ParamField>

### Filtering

Use RSQL syntax in the `q` parameter:

```
q=activo==true;comite.id==1
q=estadoActual.nombre==COMPLETADA
q=titulo=like='Research*'
```

### Response Format

Paginated responses include:

<ResponseField name="content" type="array">
  Array of result objects
</ResponseField>

<ResponseField name="pageable" type="object">
  Pagination metadata
</ResponseField>

<ResponseField name="totalElements" type="long">
  Total number of elements
</ResponseField>

<ResponseField name="totalPages" type="integer">
  Total number of pages
</ResponseField>

<ResponseField name="size" type="integer">
  Page size
</ResponseField>

<ResponseField name="number" type="integer">
  Current page number
</ResponseField>
