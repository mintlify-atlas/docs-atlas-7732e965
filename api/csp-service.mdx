## Overview

The CSP (Convocatorias, Solicitudes y Proyectos) service manages the complete lifecycle of research funding:

- **Convocatorias** - Grant calls and funding opportunities
- **Solicitudes** - Grant applications and submissions
- **Proyectos** - Funded research projects

Base URL: `/convocatorias`, `/solicitudes`, `/proyectos`

## Authentication

All endpoints require JWT authentication with appropriate role-based permissions:
- `CSP-CON-*` - Convocatoria permissions
- `CSP-SOL-*` - Solicitud permissions
- `CSP-PRO-*` - Proyecto permissions

<Note>
Permissions ending in `-C`, `-E`, `-V`, `-B`, `-R` indicate Create, Edit, View, Delete, and Reactivate respectively.
</Note>

## Convocatorias (Grant Calls)

### List Convocatorias

<CodeGroup>
```bash GET /convocatorias/investigador
curl -X GET "https://api.example.com/convocatorias/investigador?q=activo==true" \
  -H "Authorization: Bearer {token}"
```
</CodeGroup>

<ParamField query="q" type="string">
  RSQL filter query (e.g., `activo==true;titulo=like='Research'`)
</ParamField>

<ParamField query="s" type="string">
  Sort specification (e.g., `fechaPublicacion,desc`)
</ParamField>

<ParamField query="page" type="integer">
  Page number (zero-based)
</ParamField>

<ParamField query="size" type="integer" default="10">
  Page size
</ParamField>

<ResponseField name="content" type="array">
  Array of Convocatoria objects
  <Expandable title="Convocatoria object">
    <ResponseField name="id" type="long">Unique identifier</ResponseField>
    <ResponseField name="titulo" type="string">Grant call title</ResponseField>
    <ResponseField name="fechaPublicacion" type="date">Publication date</ResponseField>
    <ResponseField name="fechaProvisional" type="date">Provisional deadline</ResponseField>
    <ResponseField name="fechaConcesion" type="date">Award date</ResponseField>
    <ResponseField name="estado" type="object">Current state</ResponseField>
    <ResponseField name="unidadGestionRef" type="string">Management unit reference</ResponseField>
    <ResponseField name="modeloEjecucion" type="object">Execution model</ResponseField>
    <ResponseField name="activo" type="boolean">Active status</ResponseField>
  </Expandable>
</ResponseField>

### Get Convocatoria by ID

<CodeGroup>
```bash GET /convocatorias/{id}
curl -X GET "https://api.example.com/convocatorias/123" \
  -H "Authorization: Bearer {token}"
```
</CodeGroup>

<ParamField path="id" type="long" required>
  Convocatoria identifier
</ParamField>

### Create Convocatoria

<CodeGroup>
```bash POST /convocatorias
curl -X POST "https://api.example.com/convocatorias" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "titulo": "National Research Grant 2024",
    "fechaPublicacion": "2024-01-15",
    "unidadGestionRef": "UG-001",
    "modeloEjecucion": {"id": 1},
    "activo": true
  }'
```
</CodeGroup>

**Required Permission:** `CSP-CON-C`

### Update Convocatoria

<CodeGroup>
```bash PUT /convocatorias/{id}
curl -X PUT "https://api.example.com/convocatorias/123" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"titulo": "Updated Title"}'
```
</CodeGroup>

**Required Permission:** `CSP-CON-E`

## Solicitudes (Applications)

<CodeGroup>
```bash GET /solicitudes

  -H "Authorization: Bearer {token}"
```
</CodeGroup>

Returns applications within the user's management unit.

<ParamField query="q" type="string">
  RSQL filter query
</ParamField>

<ResponseField name="content" type="array">
  Array of Solicitud objects
  <Expandable title="Solicitud object">
    <ResponseField name="id" type="long">Unique identifier</ResponseField>
    <ResponseField name="titulo" type="string">Application title</ResponseField>
    <ResponseField name="convocatoriaId" type="long">Related grant call ID</ResponseField>
    <ResponseField name="estado" type="object">Current state</ResponseField>
    <ResponseField name="solicitanteRef" type="string">Applicant reference</ResponseField>
    <ResponseField name="creadorRef" type="string">Creator reference</ResponseField>
    <ResponseField name="activo" type="boolean">Active status</ResponseField>
  </Expandable>
</ResponseField>

### Get Solicitud by ID

<CodeGroup>

### Create Solicitud

<CodeGroup>
```bash POST /solicitudes

  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
```

</CodeGroup>

**Required Permission:** `CSP-SOL-C` or `CSP-SOL-INV-C`

<CodeGroup>

```bash PATCH /solicitudes/{id}/cambiar-estado
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"estado": "PRESENTADA"}'
```

</CodeGroup>

</ResponseField>

<ResponseField name="error" type="string">

</ResponseField>

</ResponseField>

<ResponseField name="path" type="string">

### Common Error Codes

- `400` - Bad Request: Invalid parameters or request body
- `401` - Unauthorized: Missing or invalid authentication token
- `403` - Forbidden: Insufficient permissions
- `404` - Not Found: Resource does not exist
- `409` - Conflict: Business rule violation
- `500` - Internal Server Error
