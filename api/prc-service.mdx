## Overview

The PRC (Producción Científica) service manages research output tracking and evaluation:

- **Publicaciones** - Scientific publications
- **Congresos** - Congress presentations
- **Comités Editoriales** - Editorial board memberships
- **Obras Artísticas** - Artistic works
- **Direcciones de Tesis** - Thesis supervision
- **Actividades** - Research activities

Base URL: `/producciones-cientificas`

## Authentication

All endpoints require JWT authentication with PRC-specific permissions:
- `PRC-VAL-*` - Validation permissions for managers
- `PRC-VAL-INV-ER` - Researcher validation permissions

## Publicaciones (Publications)

### List Publications

<CodeGroup>
```bash GET /producciones-cientificas/publicaciones
curl -X GET "https://api.example.com/producciones-cientificas/publicaciones?q=estado==VALIDADO" \
  -H "Authorization: Bearer {token}"
```
</CodeGroup>

Returns all publications (for managers).

**Required Permission:** `PRC-VAL-V`, `PRC-VAL-E`

<ParamField query="q" type="string">
  RSQL filter query
</ParamField>

<ResponseField name="content" type="array">
  Array of Publication objects
  <Expandable title="Publication object">
    <ResponseField name="id" type="long">Production ID</ResponseField>
    <ResponseField name="titulo" type="string">Publication title</ResponseField>
    <ResponseField name="tituloPublicacion" type="string">Journal/book title</ResponseField>
    <ResponseField name="fechaPublicacion" type="date">Publication date</ResponseField>
    <ResponseField name="tipoProduccion" type="string">Production type</ResponseField>
    <ResponseField name="estado" type="object">Validation state</ResponseField>
    <ResponseField name="epigrafe" type="string">Category</ResponseField>
  </Expandable>
</ResponseField>

### List Researcher Publications

<CodeGroup>
```bash GET /producciones-cientificas/publicaciones/investigador
curl -X GET "https://api.example.com/producciones-cientificas/publicaciones/investigador" \
  -H "Authorization: Bearer {token}"
```
</CodeGroup>

Returns publications accessible to the authenticated researcher.

**Required Permission:** `PRC-VAL-INV-ER`

## Validation Workflow

### Validate Production

<CodeGroup>
```bash PATCH /producciones-cientificas/{id}/validar
curl -X PATCH "https://api.example.com/producciones-cientificas/123/validar" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"estado": "VALIDADO"}'
```
</CodeGroup>

Validates a scientific production (manager action).

**Required Permission:** `PRC-VAL-E`

### Validate as Researcher

<CodeGroup>
```bash PATCH /producciones-cientificas/{id}/validar/investigador
curl -X PATCH "https://api.example.com/producciones-cientificas/123/validar/investigador" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"estado": "VALIDADO"}'
```
</CodeGroup>

Validates own production (researcher action).

**Required Permission:** `PRC-VAL-INV-ER`

### Reject Production

<CodeGroup>
```bash PATCH /producciones-cientificas/{id}/rechazar
curl -X PATCH "https://api.example.com/producciones-cientificas/123/rechazar" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{"estado": "RECHAZADO", "comentario": "Reason for rejection"}'
```
</CodeGroup>

**Required Permission:** `PRC-VAL-E`

### Check if Modifiable by Researcher

<CodeGroup>
```bash HEAD /producciones-cientificas/{id}/modificable/investigador
curl -I "https://api.example.com/producciones-cientificas/123/modificable/investigador" \
  -H "Authorization: Bearer {token}"
```
</CodeGroup>

Returns HTTP 200 if the researcher can modify the production, 204 otherwise.

## Common Patterns

### Validation States

Productions can be in the following states:
- `PENDIENTE` - Pending validation
- `VALIDADO` - Validated
- `RECHAZADO` - Rejected

### Filtering Examples

```
q=estado.estado==VALIDADO
q=fechaPublicacion=ge='2023-01-01'
q=tipoProduccion=='ARTICULO'
q=epigrafeCVN=like='060.010%'
```
